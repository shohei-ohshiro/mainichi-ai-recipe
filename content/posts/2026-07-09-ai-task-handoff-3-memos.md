先週、急ぎの企画書をまるごとChatGPTに投げて「あとはよろしく」とだけ伝えたら、出てきた構成が私の意図とずれていて、結局ほぼ書き直しました。私も先月まで、忙しいときほど「丸ごと任せた方が早い」と思い込んでいました。

この記事を読むと、**AIに作業を渡す前に3行だけメモを書くことで、手戻りをぐっと減らせる**ようになります。丸投げをやめろという話ではなく、渡し方をひと工夫するだけの話です。

## なぜ「丸ごと渡す」と手戻りが増えるのか

大きくて曖昧な依頼をそのまま渡すと、AIは「何を一番重視すべきか」を自分で推測しながら答えを組み立てます。推測が当たれば早いのですが、外れると、全体的にそれらしいのに肝心なところがずれた文章が返ってきます。しかも文章自体は整っているため、パッと見では「良さそう」に見えてしまい、後になって作り直しに気づくことになります。

厄介なのは、依頼した本人も「何を重視してほしいか」を自分の中で言語化していないまま投げてしまっている場合が多い点です。AIの推測がずれたのではなく、そもそも渡す側の意図が3行にもなっていなかった、というケースが実際にはよくあります。

## レシピ本体：渡す前に書く3つのメモ

やることは、AIに作業を渡す前に3つのメモを書くだけです。所要時間は5分ほどです。

**メモ1: 目的とゴール（1分）**

```
これから依頼する作業について、以下を先に確認させてください。
1. この作業が終わったとき、何ができていれば「完成」ですか
2. 読む人・使う人は誰ですか
3. 一番外してはいけないポイントは何ですか
```

まず自分がこの3つに答えられるか確認します。答えに詰まる場合は、実は自分の中でもゴールが固まっていないサインです。

**メモ2: 制約条件（2分）**

```
次の前提・制約を守って作業してください。
1. フォーマット・分量（例: 箇条書き3点まで、A4一枚など）
2. すでに決まっていて変えられないこと
3. 使ってほしくない表現・避けたい方向性
```

「すでに決まっていること」を書かずに渡すと、AIはゼロから最適解を探しにいき、社内事情と噛み合わない提案を出しがちです。制約を先に渡すほど、手戻りは減ります。

**メモ3: 作業の分解（2分）**

```
この依頼を、次の粒度に分けて順番に進めてください。
1. 背景整理
2. 論点整理
3. 構成案
4. 文章化
各ステップの終わりに、次に進んでよいか私に確認してください。
```

大きな依頼を一気に渡すと、AIは全方位に浅く答えようとします。段階ごとに区切って渡すと、途中で軌道修正でき、最後まで進んでから全部やり直す事態を避けられます。

## つまずきポイント

**その1: メモを書くだけで満足し、途中経過の確認を省略してしまう。** メモ3で「確認してください」と指定していても、忙しいときは「そのまま最後まで」と流してしまいがちです。段階ごとの確認を1回でも挟むと、ずれに早く気づけます。

**その2（効かなかったこと）: 頑張って一度に完璧な指示書を書こうとすること。** 最初から全条件を漏れなく書こうとすると、メモを書く時間の方が長くなり本末転倒です。私の場合、最初の3行は5分で書ける粗さにとどめ、AIの最初の返答を見てから制約を足す方が、結局は早く仕上がりました。「渡す前に全部決めきる」のではなく「渡した後に軌道修正する前提で、最低限の3行だけ書く」くらいがちょうどいいです。

この3行メモは、依頼が大きく、後工程に影響が広い作業ほど効果を実感しやすいです。一方、1往復で終わるような小さな確認作業にまで毎回3行書いていると、逆に手間が増えます。「作り直しになったら痛手が大きいかどうか」を線引きの基準にすると、続けやすくなります。

## まとめ：今日はこれだけ

次にAIに頼もうとしている作業を1つ選び、渡す前に「完成の条件」「守ってほしい前提」の2行だけ書いてみてください。それだけで、丸投げして後から作り直す確率はかなり下がります。
