AIに「これ、要約して」と頼んで、欲しい答えが返ってくる日と、的外れな答えしか返ってこない日がある。私も先月まで、これは運かAIの機嫌の問題だと思っていました。

この記事を読むと、**同じ質問でもAIから狙った答えを引き出す3つの書き方**が、今日から使えるようになります。

## なぜ答えの質が変わるのか

AIは指示に書かれていない前提を、それらしく勝手に埋めます。「いい感じにまとめて」と頼めば、AIなりの「いい感じ」で埋める。その「いい感じ」が自分の求めているものと一致するかは運任せです。つまり答えの質が変わるのは、AIの調子ではなく**指示に埋め込んだ前提の量**で決まります。

## 書き方1: 「誰が読むか」を先に書く

**Before**
```
このプロジェクトの進捗を要約して
```

**After**
```
このプロジェクトの進捗を、初めて聞く新人が3分で概要をつかめるように要約して。
専門用語は使わず、知らない前提で説明して。
```

読み手を書かないと、AIは書き手（あなた）向けの要約を作ります。社内でそのまま経営層に見せて、専門用語だらけで作り直した経験が私にはあります。宛先の一行を足すだけで、この手戻りはほぼ消えました。

## 書き方2: 曖昧な言葉を数字に翻訳する

**Before**
```
簡潔にまとめて
```

**After**
```
300字以内、見出しは3つまで、箇条書き中心でまとめて
```

「簡潔に」「わかりやすく」は人によって基準が違う言葉です。AIにとっても同じで、字数や項目数という数字に翻訳して初めて、狙った長さ・粒度の答えが返ってきます。

## 書き方3: 欲しい形（型）を先に指定する

**Before**
```
この会議の内容を教えて
```

**After**
```
この会議の内容を「決定事項 / 保留事項 / ToDo（担当者つき）」の3項目に分けて教えて
```

型を指定しないと、AIは文章としてまとめてしまい、後で自分が箇条書きに直す作業が発生します。欲しい構成をあらかじめ渡せば、その手直しがまるごと不要になります。

## つまずきポイント

**その1: 3つを一度に全部詰め込んで指示が長文化する。** 慣れるまでは1つずつ足すのがおすすめです。宛先だけ、数字だけ、と単体で試すと効果を体感しやすく、次第に自然と全部書けるようになります。

**その2: 「わかりやすく」を別の曖昧な言葉に置き換えてしまう。** 「簡潔に」を「シンプルに」に変えても意味は同じです。数字か具体的な条件に落とし込めているか、書いた後に見直してください。

## まとめ：今日はこれだけ

次にAIに何か頼むとき、**「誰が読むか」を一行足す**。これだけ試してみてください。同じ質問でも返ってくる答えの手直し量が変わるはずです。
